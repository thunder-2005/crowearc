Regulatory Foundations Addressed (and Not)

Direct mapping to U.S. AML rules — built / partial / gap status per requirement
Rule Element Status Evidence / Gap
31 CFR 1020.320(a) SAR program elements ✓ Built End-to-end workflow + dual approval setting + audit coverage
31 CFR 1020.320(b)(3) 30-day filing deadline (60d unknown subject) ⚠ Partial Schema fields exist but detection_date mutable; no on-screen FinCEN clock; no 60-day extension fields
31 CFR 1020.320(d) 5-year SAR retention ⚠ Partial Clock computed and surfaced; filed_date + retention_expiry_date user-mutable post-filing; legal hold doesn't pause clock
31 U.S.C. § 5318(g)(2) SAR confidentiality / no tipping-off ✗ Gap Customer profile renders SAR badge + history to any viewer; SAR reads unguarded; need-to-know not enforced
31 CFR 1010.230 CDD Final Rule / beneficial ownership ⚠ Partial UBO captured as JSON blob on customer row; cannot express layered entities or look-through
31 CFR 1010.311 CTR threshold ($10k cash) ✗ Gap No CTR pathway; $10k references only in structuring detection logic
31 CFR 501.603 OFAC compliance ⚠ Partial SDN download + adjudication built; list-version per screening not captured; PEP / UN / EU / HMT not integrated
FFIEC BSA/AML Manual Audit + SAR sections ⚠ Partial Audit coverage broad; immutability is app-only; export package materially incomplete vs. examiner expectation
SR 11-7 (OCC 2011-12) Model risk governance ✗ Gap No model registry, no validation, no monitoring; required before any LLM SAR-narrative work lands
FFIEC IT Booklet Authentication + access controls ✗ Gap Plaintext password compare; client-supplied role header; no MFA; 11 demo credentials in production bundle
